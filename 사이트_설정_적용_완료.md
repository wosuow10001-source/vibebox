# 사이트 설정 전체 페이지 적용 완료

## 문제점
- 사이트 설정(배경, 로고, 메뉴, 후원 버튼 등)이 일부 페이지에만 적용되고 다른 페이지에는 적용되지 않음
- `PageLayout`과 `SiteRenderer`가 중복으로 헤더/푸터를 렌더링하여 충돌 발생
- `/a/[slug]` 페이지에서 `PageLayout`이 이중으로 감싸져 있음

## 해결 방법

### 1. 레이아웃 구조 정리
- `app/(public)/layout.tsx`: 모든 공개 페이지에 `PageLayout` 적용 (헤더, 푸터, 배경 포함)
- `SiteRenderer`: 헤더/푸터 제거, 콘텐츠 그리드만 렌더링하도록 변경
- 각 페이지: `PageLayout` 중복 제거

### 2. 수정된 파일

#### `components/public/SiteRenderer.tsx`
- 헤더, 푸터, 배경 스타일 제거
- 콘텐츠 그리드 렌더링만 담당
- 불필요한 인터페이스 속성 제거

#### `app/(public)/a/[slug]/page.tsx`
- `PageLayout` import 및 wrapper 제거
- 레이아웃은 상위 `layout.tsx`에서 자동 적용

### 3. 적용 범위
이제 모든 페이지에 사이트 설정이 자동으로 적용됩니다:
- ✅ 메인 페이지: `/`
- ✅ 상세 페이지: `/a/[slug]` (비디오, 이미지, HTML 앱)
- ✅ 포스트 페이지: `/p/[slug]`
- ✅ 태그 페이지: `/tag/[tag]`
- ✅ 향후 추가될 모든 페이지

### 4. 테스트 방법
1. 사이트 설정 페이지에서 설정 변경 (배경, 로고, 메뉴 등)
2. 저장 후 각 페이지 확인:
   - 메인 페이지
   - 콘텐츠 상세 페이지
   - 태그 페이지
3. 모든 페이지에 동일한 헤더, 푸터, 배경이 적용되는지 확인

## 결과
- 사이트 설정이 모든 페이지에 일관되게 적용됨
- 중복 레이아웃 제거로 코드 간소화
- 새로운 페이지 추가 시 자동으로 설정 적용
