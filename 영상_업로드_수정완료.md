# 42MB 영상 업로드 문제 해결 완료

## 문제 상황
- 42MB 영상 파일(`korea_hiphop_ai_musicvedio_satan1.mp4`)을 업로드하면 ~10MB로 잘림
- 영상 플레이어에 검은 화면만 나오고 재생 안 됨

## 원인
Next.js 15의 API 라우트는 기본적으로 약 10MB 바디 크기 제한이 있어서 대용량 파일이 잘렸습니다.

## 해결 방법
**청크 업로드 시스템 구현**
- 10MB 이상 파일은 자동으로 5MB씩 조각내서 업로드
- 서버에서 조각들을 합쳐서 완전한 파일 생성
- 실시간 진행률 표시

## 수정된 파일
1. **app/api/admin/assets/upload-chunk/route.ts** (새로 생성)
   - 청크 업로드를 처리하는 API
   
2. **app/admin/upload/page.tsx**
   - 파일 크기 체크해서 자동으로 청크 업로드 선택
   - 진행률 표시 개선

3. **next.config.ts**
   - 바디 크기 제한을 2GB로 증가

## 이제 해야 할 일
1. 개발 서버 재시작: `npm run dev`
2. 관리자 페이지 → 콘텐츠 업로드
3. 42MB 영상 파일 다시 업로드
4. 메인 페이지에서 영상 썸네일 확인
5. 영상 클릭해서 재생 확인

## 예상 결과
- 42MB 파일이 완전히 업로드됨
- `public/uploads/[assetId]/index.mp4` 파일 크기가 42MB
- 메인 페이지에 영상 썸네일 표시
- 상세 페이지에서 영상 정상 재생
