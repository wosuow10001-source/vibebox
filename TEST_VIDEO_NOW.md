## 테스트 단계

### 1. 개발 서버 재시작 (필수!)

현재 실행 중인 서버를 중지하고 다시 시작하세요:

```bash
# 터미널에서 Ctrl+C로 서버 중지

# 개발 서버 재시작
npm run dev
```

### 2. 브라우저 캐시 삭제

브라우저에서:
- Windows: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

또는 개발자 도구(F12) → Network 탭 → "Disable cache" 체크

### 3. 메인 페이지 확인

http://localhost:3000 접속

**확인 사항:**
- [ ] "사탄" 제목의 카드가 보이는가?
- [ ] 카드에 파란색 배경의 플레이 아이콘(▶)이 표시되는가?
- [ ] 카드 하단에 제목이 표시되는가?

### 4. 영상 상세 페이지 확인

카드를 클릭하거나 직접 접속:
http://localhost:3000/a/sample-content

**확인 사항:**
- [ ] 페이지가 로드되는가? (404 에러가 아닌가?)
- [ ] 제목 "사탄"이 표시되는가?
- [ ] 영상 플레이어가 보이는가?
- [ ] 재생 버튼을 누르면 영상이 재생되는가?
- [ ] 전체 화면 버튼이 작동하는가?

### 5. 개발자 도구 확인

F12를 눌러 개발자 도구 열기

**Console 탭:**
- [ ] "payload must be object" 에러가 없는가?
- [ ] 다른 에러가 없는가?

**Network 탭:**
- [ ] `index.mp4` 파일이 로드되는가?
- [ ] 상태 코드가 200인가?
- [ ] 파일 크기가 10MB인가?

### 6. 직접 영상 URL 테스트

브라우저에서 직접 접속:
http://localhost:3000/uploads/asset-58b598fc-ce03-4517-ad39-46c9f2031312/index.mp4

**확인 사항:**
- [ ] 영상이 직접 재생되는가?
- [ ] 404 에러가 아닌가?

## 문제 해결

### 문제 1: 404 Not Found
**원인**: 개발 서버가 재시작되지 않음
**해결**: 서버를 완전히 중지하고 다시 시작

### 문제 2: 영상이 로드되지 않음
**원인**: 파일 경로 문제
**확인**:
```bash
# PowerShell에서 실행
Test-Path "public/uploads/asset-58b598fc-ce03-4517-ad39-46c9f2031312/index.mp4"
```
True가 나와야 함

### 문제 3: 페이지는 로드되지만 영상이 안 보임
**원인**: 브라우저 캐시
**해결**: 
1. Ctrl+Shift+R로 하드 리프레시
2. 시크릿 모드로 테스트

### 문제 4: "payload must be object" 에러
**원인**: 이전 빌드 캐시
**해결**:
```bash
# .next 폴더 삭제
Remove-Item -Recurse -Force .next
npm run dev
```
